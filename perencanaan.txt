Menurut saya, desain ini **jauh lebih bagus** daripada login biasa. Ini membuat **Coordination** terasa seperti aplikasi enterprise (ERP), bukan aplikasi CRUD.

# Konsep Login Coordination

Ada **3 jenis login**, tetapi **tetap menggunakan satu tabel users**.

Yang membedakan adalah **cara autentikasi dan hak akses**.

```
Coordination

├── Login Event Owner
├── Login Ketua Divisi
└── Login Anggota Divisi
```

Jangan membuat 3 tabel login.

## 💡 Saran Terakhir (Agar Sangat Enterprise)

Saya menyarankan **mengganti istilah "Kode Akses" menjadi "Passphrase" atau "Access Passphrase"** untuk login Ketua Divisi dan Anggota. Selain terdengar lebih profesional, istilah ini membedakan autentikasi internal event dari password akun utama. Dengan kombinasi **Officer ID/Member ID + Passphrase + Event Token + OTP + Trusted Device**, Coordination akan memiliki autentikasi berlapis yang kuat tanpa mengorbankan pengalaman pengguna.
